# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

Counter-Strike: Source Community Edition — a community port of the 2007 CS:S leak onto
the **Source SDK 2013 Multiplayer** engine. Pre-alpha. C++, Valve Source engine codebase.
Based on SourcePlusPlus's CS:S port with a custom main menu (GameUI2) and FMOD audio.

## Build

Two steps: generate projects with VPC, then compile. Binaries land in
`mp/game/community/bin`. No test suite.

### Linux (32-bit; the actively-supported build here)

Prereq (one-time): `sudo apt install gcc-multilib g++-multilib`. A fresh checkout loses
executable bits, so restore them first:
```
cd mp/src
chmod +x devtools/bin/vpc* devtools/bin/linux/ccache devtools/gendbg.sh creategameprojects createallprojects
```
Generate + compile:
```
devtools/bin/vpc_linux /community +game /mksln Game-COMMUNITY   # writes Game-COMMUNITY.mak
make -f Game-COMMUNITY.mak                                       # or: ... server | client | gameui2
```
Produces `client.so`, `server.so`, `gameui2.so` (+ `_srv` dedicated-server variants).

Notes: the code predates modern gcc, so `devtools/makefile_base_posix.mak` builds with
`-std=gnu++0x -fpermissive -fabi-compat-version=2` — many warnings are expected. Parallel
make interleaves errors; strip ANSI codes (`sed 's/\x1b\[[0-9;]*m//g'`) before grepping logs.
**FMOD background music is Windows-only** (no Linux FMOD lib) — its files are `[$WIN32]`-guarded
in `client_community.vpc`, so the Linux client builds without it.

### Windows

`creategameprojects.bat` (or `createallprojects.bat` for the map/build tools too) →
`devtools\vpc.exe /community +game /mksln Game-COMMUNITY.sln`. Open the `.sln` in Visual
Studio 2022 (MSVC v143 + Windows SDK), build Debug/Release →
`client.dll`/`server.dll`/`gameui2.dll`. Shaders:
`materialsystem/stdshaders/buildcommunityshaders.bat`.

`./createallprojects` adds the map/build tools (vbsp, vrad, etc.) via `+everything`.

## Run

Requires these Steam apps installed for content mounting (see `gameinfo.txt`):
Source SDK Base 2013 MP (243750), Counter-Strike Source (240), Half-Life 2 (220).

- Linux: launch the SDK Base 2013 MP engine pointed at the mod:
  `<SteamLibrary>/Source SDK Base 2013 Multiplayer/hl2.sh -game <repo>/mp/game/community -dev +sv_cheats 1 -console`
- Windows: `mp/game/community/run_mod.bat` (also `run_mod_debug.bat`, `run_mod_tools.bat`),
  which launches `hl2.exe` from the SDK Base with `-game <community dir>`.

## Architecture

Standard Source-engine three-target layout. Source lives in `mp/src/`; shipped game
content (maps, materials, scripts, `gameinfo.txt`) in `mp/game/community/`.

- **`mp/src/game/client/`** → `client.dll`. Rendering, HUD, prediction, UI.
- **`mp/src/game/server/`** → `server.dll`. Game rules, entities, AI.
- **`mp/src/game/shared/`** → compiled into BOTH client and server. A file here is
  built twice; behavior is switched with the `CLIENT_DLL` vs `GAME_DLL` preprocessor
  macros. Shared entities use the `C_`-prefix (client) / no-prefix (server) convention
  with predicted/networked vars. Edit shared code with both sides in mind.
- **`mp/src/game/gameui2/`** → `gameui2.dll`. NicolasDe's custom main-menu replacement,
  built on VGUI2 (`vgui2d/` low-level widgets, `buttons/`, `panels/`). Loaded via the
  `GAMEUI2` define and `igameui2.h` interface.

CS-specific gameplay lives in `cstrike/` subdirs (`game/client/cstrike`,
`game/server/cstrike`, `game/shared/cstrike` — players, weapons, bomb, hostages, game
rules). Community-specific additions live in `community/` subdirs. Other notable server
subsystems: `game/server/fmod/` (FMOD audio) and `game/server/NavMesh/` (bot navigation).

Engine support libraries (`tier0`, `tier1`, `vstdlib`, `mathlib`, `vgui2`,
`materialsystem`, `public/`) and build tools (`utils/`) are stock Source SDK — change
only when necessary.

## VPC build system

Projects are defined in `.vpc`/`.vgc` scripts, not Makefiles/solutions directly — those
are generated by VPC. To add/remove source files from a build, edit the relevant `.vpc`
and regenerate, do NOT hand-edit generated solutions/Makefiles.

- Project → script mapping: `mp/src/vpc_scripts/projects.vgc`.
- Per-target scripts: `game/client/client_community.vpc`, `game/server/server_community.vpc`,
  `game/gameui2/gameui2_base.vpc`. Shared base configs in `mp/src/vpc_scripts/`.
- Community builds are gated by the `$COMMUNITY` VPC condition and the `/community` flag.
- Key client defines (`client_community.vpc`): `CSTRIKE_DLL`, `GAMEUI2`,
  `COMMUNITY_CLIENT_DLL`, `GLOWS_ENABLE`.

## Known issues — code-audited 2026-06-05

The README's original bug list was audited against the source; **most claims did not
hold up** (already fixed or mis-diagnosed). Do NOT trust the old list — verify against
code. See README.md "Bug status" for evidence/line refs.

**Fixed since the audit** (see README "Fixed since the audit"):
- Ragdoll bullet-impact crash — null bone-matrix guard in `bone_setup.cpp`
  (`SweepBoxToStudio`/`TraceToStudio`) + `fx_impact.cpp` (working tree).
- Bot use-after-free — `m_attacker` raw pointer → `CHandle` (`c5c7d63`).
- Round-timer spawn crash (`swprintf`) + GameUI2 input yield (`cdfced7`).
- Friendly target-ID literal `%s` → health (`7115f70`).

Summary of remaining verdicts:

- **Animation "stuck on first frame" — FALSE.** The real CS:S `CCSPlayerAnimState`
  (`game/shared/cstrike/cs_playeranimstate.cpp:57`) is used with `LEGANIM_9WAY` on both
  client and server. `CBasePlayerAnimState` is its *base class*, not the broken HL2MP
  animstate. This is working multiplayer animation — don't "rewrite" it.
- **Rounds don't end at 0:00 — FALSE for normal maps.** `BombRoundEndCheck()`
  (`cs_gamerules.cpp:1728`) + `CheckRoundTimeExpired()` (`:2725`) handle it; a planted
  bomb intentionally runs past 0:00. Real latent gap: objective-less maps lost the draw
  fallback ("get rid of round draws", `:2733`).
- **Flashbang fade — render code is correct** (`cs_view_scene.cpp:168`). If it ever
  sticks, suspect the networked `m_flFlashBangTime`/duration, not the renderer.
- **Dynamic crosshair cvars — all present/used** in `weapon_csbase.cpp DrawCrosshair()`.
- **Classmenu lighting — unconfirmed.** Candidate only: `basemodelpanel.cpp:601`
  (`white[6]` ambient = 0.4 gray, dim/misnamed).
- **Bots may crash — inconclusive** (not provable statically; needs a repro).
- **Prop physics — not yet audited.**

Before assuming a bug is real or newly introduced, check the code path first — the
historical README was unreliable.
